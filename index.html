MongoDB for .NET Developers - CRUD Operations

这是M101N: MongoDB for .NET Developers的第二周课程，介绍了在Mongo Shell和.NET里CRUD的操作。

CRUD含义

  • CRUD代表数据库的四种操作： Create, Read, Update, Delete

  • 在MongoDB中这四种操作分别对应叫做： Insert, Find, Update, Remove

  • 与之对比，在SQL中分别对应： Insert, Select, Update, Delete

  • MongoDB里的CRUD和SQL有本质的区别： SQL是独立的语言，而MongoDB的CRUD操作需要借助其他编程语言来实现的，如Java，C#等

BSON介绍

BSON是Binary JSON的简称，是一种类JSON的二进制形式的存储格式。和JSON类似，BSON也支持内嵌的文档和数组对象。同时，BSON含有更多JSON中没有的数据类型，如ObjectId和datetime等，所以我们可以把BSON看做JSON的父集，具体参考BSON Spec文档。Mongo Shell和其他所有能与MongoDB对话的driver都支持所有BSON的数据类型。BSON对象的一个例子：

I. MongoDB几个重要基本概念

  1. database(数据库)： 一个MongoDB服务器通常有多个数据库，每个数据库是集合的容器。在Mongo Shell中通过use <db>来选择数据库。

  2. collections(集合)： 相当于RDBMS(关系数据库)里Table(表)的概念，集合由不同的文档组成。

  3. documents(文档): 大致相当于RDBMS中Row/Record(行)的概念，不同的是，同一集合中，不同的文档不要求遵循相同的数据结构和字段(Schema)。每一个文档由多个键值组成(field-and-value pairs)。每个键对应的值可以是任意的BSON数据类型。比如：

Find操作 (Read)

1. findOne()

返回集合中满足查询条件的一个文档。如果存在多个文档满足条件，则根据文档存储的顺序返回第一个。可以带两个参数，第一个参数query，相当于RDBMS中的where语句，第二个参数projection，规定返回哪些字段，如果不提供则返回所有字段。

注意”_id”默认一定返回，如果不需要”_id”字段，必须在第二个参数projection里把”_id”设为false。
